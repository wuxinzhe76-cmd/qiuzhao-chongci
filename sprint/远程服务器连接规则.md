## 二、网络代理配置（关键）

### 代理状态
- Clash 代理运行中：`clash -d /etc/clash`
- 代理端口：7890（mixed-port，支持 HTTP 和 SOCKS5）

### 🔴 核心问题
环境变量未设置 `http_proxy` / `https_proxy`，导致 `pip install`、`curl` 等命令无法访问外网。

### 解决方案
**执行任何需要联网的命令前，必须先设置代理：**

```bash
export http_proxy=http://127.0.0.1:7890 https_proxy=http://127.0.0.1:7890
```

### 代理测试
```bash
curl -x http://127.0.0.1:7890 https://pypi.org
# 返回 200 表示代理正常
```

## 九、注意事项

1. **内存紧张**：15GB 内存已用 10GB，可用仅 5.2GB。测试时按需启动中间件，不要同时启动所有服务。
2. **无 Kafka/ClickHouse**：服务器上没有 Kafka 和 ClickHouse。可观测性系统测试需要单独搭建或跳过。
3. **消息队列是 RabbitMQ**：不是 Kafka。面试报告异步生成走 RabbitMQ，记忆升级也是 RabbitMQ。
4. **代理必须手动设置**：Clash 运行但环境变量未配置，pip/curl 访问外网前必须 export proxy。
5. **应用内部端口**：interview-backend 在 8080 但未映射到宿主机，必须通过 Nginx 端口 80 访问。
6. **单机部署**：所有服务单机部署，测试时模拟并发用户即可，不要做分布式测试。
7. **pip 系统包**：Python 3.12 需要 `--break-system-packages` 标志安装包。
