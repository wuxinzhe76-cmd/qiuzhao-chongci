## 模块一：GraphRAG 基础概念与核心原理

**目标**：搞清楚 GraphRAG 是什么、为什么需要、与传统 RAG 的区别

### 知识点清单

| 编号 | 知识点 | 关键内容 |
|------|--------|---------|
| 1.1 | GraphRAG 定义 | 官方定义、核心思想（图结构+向量检索融合） |
| 1.2 | 传统 RAG 三大瓶颈 | 全局性问题无法处理、多跳推理断裂、语义孤岛效应 |
| 1.3 | GraphRAG 五大核心组件 | 实体识别(NER) -> 关系抽取(RE) -> 图谱构建(KG Build) -> 图检索(Graph) -> 答案生成(Answer Gen) |
| 1.4 | GraphRAG vs 传统 RAG vs 混合 RAG | 检索方式/推理能力/可解释性/索引成本/查询延迟对比 |
| 1.5 | 四种实现模式（Neo4j 分类法） | Type1 图增强向量 / Type2 向量引导图 / Type3 图优先 / Type4 完整 GraphRAG |

### 面试高频考点
- GraphRAG 和普通 RAG 有什么区别？
- 什么场景该用 GraphRAG，什么场景不该用？
- GraphRAG 的核心组件有哪些？

## 模块二：GraphRAG 索引流水线（Indexing Pipeline）

**目标**：掌握 GraphRAG 索引构建的完整流程，这是面试深挖的重点

### 知识点清单

| 编号 | 知识点 | 关键内容 |
|------|--------|---------|
| 2.1 | 文本分块策略 | 固定分块 / 递归分块 / 语义分块，GraphRAG 默认 1200 tokens |
| 2.2 | LLM 驱动的实体关系抽取 | Prompt 工程、Gleaning 机制（多轮抽取提升召回） |
| 2.3 | 知识图谱构建 | Neo4j 属性图模型、节点/边/属性设计、唯一约束 |
| 2.4 | Leiden 社区发现算法 | 分层聚类原理、社区摘要生成、延迟优化 60% 的原理 |
| 2.5 | 向量索引与图索引 | 向量嵌入、图索引、双路索引协同 |

### 面试高频考点
- GraphRAG 的索引流水线是怎样的？
- Leiden 社区发现算法是什么？怎么用的？

## 模块三：GraphRAG 检索策略

**目标**：掌握 GraphRAG 的多种检索模式及其适用场景

### 知识点清单

| 编号 | 知识点 | 关键内容 |
|------|--------|---------|
| 3.1 | Local Search（局部检索） | 实体邻域子图检索，适合精确事实查询 |
| 3.2 | Global Search（全局检索） | 社区摘要层次化检索，适合全局分析 |
| 3.3 | DRIFT Search（混合检索） | Local + Global 融合，兼顾精度和全局视角 |
| 3.4 | 混合检索路由 | 意图识别 -> 动态路由到最优检索策略 |

### 面试高频考点
- Local Search 和 Global Search 的区别？
- DRIFT Search 是什么？
- 如何根据查询意图选择检索策略？

## 模块五：企业级增强能力（生产级关键）

**目标**：掌握让 GraphRAG 达到生产级可用性的增强手段

### 知识点清单

| 编号 | 知识点 | 关键内容 |
|------|--------|---------|
| 5.1 | 限流机制 | 令牌桶算法 + 滑动窗口，保护 LLM API 不被打爆 |
| 5.2 | 熔断与降级 | Circuit Breaker 模式，故障时降级到缓存或简单 RAG |
| 5.3 | 重试与超时策略 | 指数退避重试、超时分级（向量检索 2s / 图遍历 5s / LLM 30s） |
| 5.4 | 缓存策略 | 语义缓存（相似问题复用）、社区摘要预加载 |
| 5.5 | 异步任务管理 | 长耗时任务（视频生成）的异步调度 + 状态轮询 + 回调 |
| 5.6 | 监控与告警 | Prometheus + Grafana，监控延迟/错误率/Token 消耗 |

### 面试高频考点
- LLM API 被限流了怎么办？
- GraphRAG 服务挂了怎么降级？
- 怎么做语义缓存？
